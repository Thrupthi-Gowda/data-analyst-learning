# Day 8 – SQL Constraints

## Overview
On Day 8, I learned how to apply constraints in SQL to ensure data accuracy and integrity.

## Topics Covered
- PRIMARY KEY
- NOT NULL
- UNIQUE
- DEFAULT

## Description
Constraints are rules applied to columns to control the type of data that can be inserted.

## Key Learnings
- PRIMARY KEY uniquely identifies each record.
- NOT NULL ensures a column cannot have empty values.
- UNIQUE prevents duplicate values.
- DEFAULT assigns a value when none is provided.
